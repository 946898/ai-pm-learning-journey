# 一、MCP：模型的“工具包”

## 1.1MCP是什么？

Model Context Protocol（MCP）是一个开放协议，用于在 AI 应用（如 Claude Desktop）和外部工具/数据源之间建立标准化连接。

## 1.2核心概念

| 概念         | 说明                                           |
| :----------- | :--------------------------------------------- |
| **Server**   | 提供工具/资源的服务端                          |
| **Tool**     | 可被 AI 调用的函数（如 `check_supplier_risk`） |
| **Resource** | 可被 AI 读取的数据（如文件、数据库）           |
| **Prompt**   | 预定义的提示词模板                             |

## 1.3如何工作？

1. 模型根据问题，判断是否需要及需要调用哪个工具。
2. 模型生成符合规范的调用指令。
3. 平台执行调用，并将结果返回给模型。
4. 模型将结果整合进回复中。

- 实战场景
  - “今天纽约的天气怎么样？帮我推荐合适的穿搭。”
  - 复合能力过程：
    - 识别意图：需要实时天气数据
    - 路由：调用MCP-Weather工具，获取纽约当前的温度、湿度、降水概率
    - 结果整合：基于天气数据+本身知识（服装搭配），生成穿搭建议
  - 其他MCP工具示例
    - MCP-SQL：直接查询公司数据库
    - MCP-Google-Search：执行联网搜索
    - MCP-Excel：自动生成或处理电子表格

## 二、与 LangGraph 的对比

- LangGraph：面向 Agent 工作流编排（内部流程）
- MCP：面向 Agent 与外部工具/数据源的标准化通信（外部接口）
